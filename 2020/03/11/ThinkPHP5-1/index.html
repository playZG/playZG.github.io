ThinkPHP5.1入门

框架简介

１．TP框架是免费开源的、轻量级的、简单快速且敏捷的PHP框架

２．可以免费使用TP框架，甚至也可以将你的项目商用

３．ThinkPHP5.1要求PHP版本是5.6+以上(最新版本是7.x)

４．以下我们会采用7.x来运行TP5

安装

官网不提供软件包下载]，但是支持Composer方式下载，安装Composer可以去这里，入门指南里面有全局安装。

URL解析

  • ThinkPHP 框架非常多的操作都是通过 URL 来实现的；

  • http://serverName/index.php/模块/控制器/操作/参数/值…；

  • index.php 为入口文件，在 public 目录内的 index.php 文件；

  • 模块在 application 目录下默认有一个 index 目录，这就是一个模块；

  • 而在 index 目录下有一个 controller 控制器目录的 Index.php 控制器；

  • Index.php 控制器的类名也必须是 class Index，否则错误；

  • 而操作就是控制器 class Index 里面的方法，比如：index 或 hello；

  • 那么完整形式为：public/index.php/index/index/index

为了方便理解，我们自己创建一个

在 application 目录下创建一个 test 目录（模块）

在 test 模块下创建控制器目录 controller，并在下面创建 Abc.php（控制器），创建一个Abc类，在里面创建一个playzg方法，URL完整参数：public/index.php/test/abc/playzg

模块设计

目录结构

  • ThinkPHP5.1默认是多模块架构，也可以设置为单模块操作

  • 所有模块的命名空间以app这三个字母作为根命名空间(可通过环境变量更改)

  • 模块下的类库文件命名空间统一为：app\模块名，比如：app\index\controller\Index

  • 如果只有一个zg模块时，可以绑定这个模块，在public\index.php的文件，追加一个方法

    Controller::get(‘app’)->bind(‘zg’)->run()->send();

    这个时候URL就变成了：public/abc/playzg，多模块时，其他的将无法访问

  • 如果只有一个模块，只有一个控制器，那我们可以改写追加方法：Container::get(‘app’)->bind(‘zg/abc’)->run()->send()

    此时URL调用就变成了：public/playzg，得到了非常简化：但其他控制器无法访问

空模块

  • 可以通过环境变量设置空目录，将不存在的目录统一指向指定目录
  • 在config的目录下的app.php修改：empty_module =>’index’
  • 空模块只有在多模块开启，且没有绑定模块的情况下生效

单一模块

如果应用只有一个模块，那就可以直接设置单模块

在config目录下的app.php修改：app_multi_module =>false

这时URL地址：public/index/one,即：控制器/操作

单一模块的命名空间也变更为：app/controller

控制器定义

控制器定义

  • 控制器controller，控制器文件存放在controller目录下
  • 类名和文件名大小写保持一致，采用驼峰命名
  • 如果只有一个单词，首字母大写，比如：class Index，URL访问时直接public/index即可
  • 如果创建的是双字母组合，比如：class HelloWorld，URL访问必须为：public/hello_world
  • 如果想要原样的方式访问URL，需要关闭配置文件中自动转换：url_convert =>false，此时URL访问可以为：public/HelloWorld
  • 如果想要改变根命名空间app，可以在根目录下创建.env文件，然后写上配对的键值对即可，app_namespace=application

渲染输出

  • ThinkPHP直接采用方法内return返回的方式直接就输出

  • 使用json输出，直接采用json函数

    $data = array(‘a’=>1,’b’=>2,’c’=>30);

    return json($data);

  • 使用view输出模板，开启错误提示，可知道如何创建模板：return view();

  • 如果继承了基类控制器，那么可以定义控制器初始化方法：initialize()

  • initialize()方法会在调用控制器方法之前执行

    protected function initialize(){

        echo ‘init’;

    }

    initialize()方法不需要任何返回值，输出用PHP方式，return无效

  • ThinkPHP中initialize()和construct()这两个函数都可以理解为构造函数，initialize()是ThinkPHP特有的，construct()是PHP中的，但是这两个是有区别的。

默认的等待时间都是3秒

success和error方法都可以对应的模板，默认的设置是两个方法对应的模板都是：’thinkphp/tpl/dispatch_jump.tpl’

我们可以改变默认的模板：

  1. 默认错误跳转对应的模板文件
  2. ‘dispatch_error_tmpl’ => ‘../application/tpl/dispatch_jump.tpl’
  3. 默认成功跳转对应的模板文件
  4. ‘dispatch_success_tmpl’ => ‘../application/tpl/dispatch_jump.tpl’

也可以使用项目内部的模板文件

  1. 默认错误跳转对应的模板文件

  2. ‘dispatch_error_tmpl’ => ‘public/error’

  3. 默认成功跳转对应的模板文件

  4. ‘dispatch_success_tmpl’ => ‘public/success’

连接数据库
