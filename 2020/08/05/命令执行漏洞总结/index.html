命令执行漏洞

命令执行漏洞是指攻击者可以随意执行系统命令，命令执行漏洞不仅存在于B/S架构中，在C/S架构中也常常遇到。

PHP命令执行

PHP提供了部分函数用来执行外部应用程序，如：php代码相关： eval()，assert()，preg_replace，call_user_func()，call_user_func_array()，create_function，array_map()。系统相关：system()，passthru()，exec()，pcntl_exec()，shell_exec()，popen()，proc_open()，ob_start()

JAVA

在Java SE中，存在Runtime类，该类中提供了exec方法用在单独的进行中执行指定的字符串命令

管道符

Linux

常见的管道符：|,||,&&,&,;

|：直接执行后面的语句

||：如果前面的命令是错误的才会执行后面的命令，否则只会执行前面的命令

&：前后命令都会执行

&&：只有前面的命令执行了后面的命令才能执行

;：这个应该是和&一样的，前后命令都会执行

Windows

windows跟Linux相比，少了；，其他用法都是和Linux一样的

空格过滤

在很多CTF中经常会过滤空格，但是这种情况是可以绕过的

常用的绕过方式有：$IFS，${IFS}，$IFS$1，<>，<

编码绕过

base64编码绕过

单引号、双引号、反斜线

$1、$2等和$@

$后面跟的数字好像只能到9，两位数或者更多就报错

黑名单

假设过滤了cat，拼接一下是可以绕过的

我们还可以使用具有相同功能的命令

通配符

前面的几个通配符/???应该代表的就是/bin,这里采用的应该是绝对路径

内联执行绕过

命令和$(命令)都是执行命令的方式，内联就是将反引号内命令的输出作为输入执行

长度限制

Linux：>和>>

>：将结果添加到指定文件，会覆盖原文件内容

>>：将结果添加到指定文件，不会覆盖原文件内容，内容追加到原文件末尾

命令换行

在Linux中，我们通过在没有写完的命令后面加\，可以将一条命令写多行，很多时候会配合>和>>一起使用

例题：

[GXYCTF2019]Ping Ping Ping

看形式应该是命令执行了，输入127.0.0.1发现确实存在，输入127.0.0.1　%26%26 ls(%26%26是&&)也不行，在输入127.0.0.1　%7c%7c ls(%7c%7c是||)，发现可以执行，并可以看到有两个文件index.php,flag.php

再用上面那条命令127.0.0.1　%7c%7c cat flag.php发现空格被过滤了

空格可以使用$IFS ${IFS} $IFS$1绕过，但是还是不能读取flag.php,读取一点index.php

index.php

这题去看有好多解法：

解法一

因为过滤了flag和hash，但是sh没有过滤

payload：echo$IFS$1Y2F0IGZsYWcucGhw|base64$IFS$1-d|sh

解法二

payload：cat$IFS$1`ls`

解法三

payload：a=g;cat$IFS$1fla$a.php
