PHP序列化和反序列化

简单讲：系列化：将对象转换为字符串，反序列化：就是字符串转换为对象

PHP反序列化漏洞，也叫php对象注入（PHP Object Injection）。 PHP中序列化对应的函数是serialize()、反序列化函数unserialize()

反序列化本身是没有问题的，但是当反序列化参数可控，魔术方法可以利用的时候，漏洞就产生了

序列化字符串

解释一下反序列化后的字符串代表什么意思

O:1:"A":2：O代表是对象、１代表类名长度、Ａ是类的名字

s:4:"name";s:6:"playzg"：s代表字符串、４属性名长度、name属性名、s代表字符串、６属性值长度、playzg属性值

我们序列化后的字符串是这样的O:1:"a":2:{s:5:"uname";s:6:"playzg";s:8:"password";s:1:"a";}

反序列化字符串都是以一";}结束的，所以如果我们把";}带入需要反序列化的字符串中（除了结尾处），就能让反序列化提前闭合结束，后面的内容就不起作用了。

当读取字符长短是根据前面的s确定，s的值要跟参数的值长度一样，要不然就会报错了，比如：s:5:"playzg",这样就会报错，因为s的值是5,他就会向后读取5个字符，但是后面还有一个字符g,而正常的语法是需要用";去闭合当前的变量，所以就会出错，这个地方时先序列化然后又经过filter()函数过滤的，所以s的值并不会发生变化，但是Firebasky已经被替换成了Firebaskyup,也就是说这个地方逃逸除了两个字符，我们要通过";s:8:"password";s:5:"zgzhi";}闭合并改变password的值，这段字符长度是30,所以说我们要用15个Firebasky逃逸出30个字符

注意

变量是private类型的话，序列化出来的结果形式：\x00 + 类名 + \x00 + 变量名

变量是protected类型的话，序列化出来的结果形式：\x00 + * + \x00 + 变量名

把”\x00”替换为”%00”也可起到同样效果；
