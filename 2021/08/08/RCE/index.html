ThinkPHP5.0.x~5.0.23 RCE

TP5
发布日期:   2021-08-08
文章字数:   2.4k

ThinkPHP5.0.x~5.0.23远程代码执行

漏洞描述

ThinkPHP5.0在核心代码中实现了表单请求类型伪装的功能，该功能利用$_POST['_method']变量来传递真实的请求方法，当攻击者设置$_POST['_method']=__construct时，Request类的method方法便会将该类的变量进行覆盖，攻击者利用该方式将filter变量覆盖为system等函数名，当内部进行参数过滤时便会进行执行任意命令。

影响版本

版本：5.0~5.0.23

复现

搭建 : PHPStudy

服务器：apache

PHP：7.0.9

TP版本：ThinkPHP 5.0.22

POC

利用方式一

/thinkphp5/public/index.php?s=captcha&cmd=whoami
_method=__construct&filter=system&method=get

利用方式二

/thinkphp5/public/index.php?s=captcha&cmd=whoami
_method=__construct&filter=system&method=get&server[REQUEST_METHOD]=id

漏洞分析

漏洞出现在call_user_func()函数，并且两个参数均可控，造成命令执行。可以反向分析学习一下怎么利用到call_user_func()函数完成命令执行。

call_user_func($filter, $value)：中的filter和value都是参数传入的，看一下那个地方调用了filterValue(&$value, $key, $filters)这个函数。

在cookie函数中和input()函数中被调用，这里着重看一下input()函数。

filterValue($data, $name, $filter)中$data、$filter也都是当作参数传入的input($data = [], $name = '', $default = null, $filter = '')，但是$filter经过了getFilter($filter, $default)函数的处理，看一下getFilter()函数的作用。

当$filter存在的时候$filter = $this->filter，经过处理之后返回，回到input()函数，然后就是get()函数中调用了input()函数。

如果$this->get为空的话，$this->get = $_GET，也就是这个参数可控，这个参数也就是input()函数中的$data参数->filterValue()函数中的$value参数->call_user_func()函数中的$value。

在param函数中调用了get函数

call_user_func()函数中的$filter要利用到method()

特别注意525行，先从配置文件中得到var_method的值(var_method=_mothod)，然后POST方式获取变量_mothod的值，作为动态函数调用。因为这个是在Request类下面，所以可以动态调用这个类下面的所有函数。我们是想利用这个去控制call_user_func()函数中的$filter，并且在getFilter()函数中看到了当$filter存在的时候$filter = $this->filter，这里可以利用Request类的__construct()函数去覆盖 $this->filter。

Request类的__construct()函数，该方法把参数遍历并赋值，所以说这两个参数均可空，可以达到RCE。

漏洞利用

通过分析可知，最后是调用的Request类method()和param()函数，看一下是那两个地方调用的这两个函数。

然后层层返回到Route.php中的checkRoute()方法再到App.php中的routeCheck()方法，再到App.php中的run()方法，在run()方法中进入了exec方法，此时$dispatch的值为就是上图的$result，跟进exec方法

在run()->exec()->Request::instance()->param()， 当exec()方法进入到method分支时调用了Request类的param()方法

param()方法中调用了get()方法，从这里开始就跟刚开始分析的利用的顺序差不多了

get()方法中调用了input()方法.

input()方法中调用getFilter()函数的处理，然后又调用了filterValue()，在filterValue()中调用call_user_func()函数完成命令执行。

如果开启debug的时候，run()函数就会直接进入Request类中的param()函数，不需要再进入exec()函数，也就没有对captcha的依赖了

在利用方式二中可以看到有个参数server[REQUEST_METHOD]=id，这种利用方式跟上面的区别就在于param()函数调用method()函数时候传递了参数为true

当method()的形参为true时，调用server()方法，该方法中调用了inpout()函数，input()方法中调用getFilter()函数的处理，然后又调用了filterValue()，在filterValue()中调用call_user_func()函数完成命令执行。
